<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| upstream-to-public-api | Upstream pressure/failure is translated into public client JSON or SSE behavior. |
| upstream-stream-to-downstream-stream | Upstream streaming failures can occur before or after downstream content is emitted. |
| handler-to-request-log | Retry attempts and exhaustion reasons are written to operational evidence. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-11 | Denial of Service | Opus 4.7 retry loops | mitigate | Enforce `defaultOpus47MaxAttempts` and request wait budget in all Opus 4.7 protocol paths. |
| T-04-12 | Spoofing | pressure exhaustion responses | mitigate | Return retryable pressure metadata instead of stable fake success when budgets are exhausted. |
| T-04-13 | Repudiation | stream failure logging | mitigate | Append request-log attempt/failure evidence for budget exhaustion and post-start stream failures. |
| T-04-14 | Tampering | streaming retry control | mitigate | Test `streamStarted` guards so a started downstream stream cannot be replayed on another account. |
| T-04-SC | Tampering | package installs | accept | No npm/pip/cargo package installs are planned in this phase plan. |
</threat_model>
