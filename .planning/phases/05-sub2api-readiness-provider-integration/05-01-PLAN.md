# 05-01 Plan: Configure and Query Kiro-Go Readiness in sub2api

## Goal

sub2api can configure a Kiro-Go readiness provider and safely query/cache the Phase 4 readiness contract for an effective upstream model.

## Tasks

1. Add gateway scheduling config fields for Kiro-Go readiness provider: enabled, endpoint/base URL, timeout, healthy/degraded/blocked/error TTLs, fail mode, model allow/match list, and optional temp-unschedule behavior.
2. Add defaults and validation in config loading.
3. Implement a nil-safe readiness provider/client/cache in `backend/internal/service`.
4. Parse Kiro-Go readiness response fields needed by scheduling: status, safeConcurrency, retryAfterSeconds, reasons, contract/version/provider identifiers where present.
5. Add unit tests for disabled mode, status parsing, cache hit/miss TTLs, fail-open/fail-closed, and model matching.

## Acceptance

- Provider is disabled by default.
- No credentials are read or logged.
- Config supports endpoint, timeout, TTLs, fail mode, and model matching.
- Tests prove cache and fail-mode behavior.
