# Phase 01 Plan 03: Real UAT Evidence Bundle

<objective>
Create a repeatable Phase 1 UAT harness that captures real API, readiness, request-log, SSE, and optional `/www/sub2api` downstream evidence without storing secrets or faking PASS when the environment or live upstream blocks verification.
</objective>

<tasks>
<task id="01-03-01" type="auto">
  <title>Create UAT harness</title>
  <action>
    Add Node scripts and README under `docs/superpowers/uat/phase1-claude-code-official-compatibility/`.
    The harness must read credentials from environment variables only and write run artifacts under `runs/<run-id>/`.
  </action>
  <acceptance_criteria>
    <criterion>Scripts do not require third-party npm dependencies.</criterion>
    <criterion>Missing credentials produce `BLOCKED_BY_ENV`, not false PASS.</criterion>
  </acceptance_criteria>
</task>

<task id="01-03-02" type="auto">
  <title>Capture SSE and downstream summaries</title>
  <action>

  </action>
  <acceptance_criteria>
    <criterion>Parser can read saved SSE files and emit JSON summaries.</criterion>
  </acceptance_criteria>
</task>

<task id="01-03-03" type="auto">
  <title>Write initial UAT result template</title>
  <action>
    Add `UAT-RESULT.md` explaining required evidence, PASS/PARTIAL/FAIL rules, and secret handling.
  </action>
  <acceptance_criteria>
    <criterion>Template requires API/log/readiness/screenshot/sub2api evidence alignment before PASS.</criterion>
  </acceptance_criteria>
</task>
</tasks>
